# 高德工具目录：慢旅游规划使用说明

本文件用于帮助模型选择正确的高德工具。高德是"地理事实层"，不要把它当作主观评价来源。评论、攻略、避坑和体验感应来自知乎等内容源。

## 发现与标准化工具

### `amap_input_tips`

用于自动补全、纠错、模糊地名、昵称、简称、用户输入一半的地点名。若还没有精确 POI ID，优先使用它获取候选。

参数：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `keywords` | string | 是 | 用户正在输入的关键词，例如"外滩"、"迪士尼"、"成都熊猫" |
| `type` | string | 否 | POI 类型编码或类型名称，多个用 `\|` 分隔，用于限定结果类型 |
| `location` | string | 否 | 当前位置坐标 `lng,lat`，用于提升提示相关性（就近推荐） |
| `city` | string | 否 | 城市名、citycode 或 adcode，限定城市范围 |
| `citylimit` | bool | 否 | 是否仅返回指定城市数据 |
| `datatype` | string | 否 | 返回数据类型：`all`（含公交等）或 `poi`，默认 `all` |

适合场景：
- 用户说"IFS 熊猫""成都熊猫""外滩附近那个美术馆"等不完整名称。
- 知乎博主提到昵称或口语化地点名。
- 调用 `amap_poi_detail` 前需要候选 POI ID。

注意：不要把它作为最终事实来源。需要继续用详情查询或关键词搜索确认。

### `amap_poi_keyword_search`

用于按关键词或 POI 类型做广泛候选发现。

参数：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `keywords` | string | 条件必填 | 搜索关键词，例如"景点"、"餐厅"、"酒店"、"博物馆"；keywords 和 types 至少填写一个 |
| `types` | string | 条件必填 | POI 类型编码或名称，多个用 `\|` 分隔；keywords 和 types 至少填写一个 |
| `city` | string | 否 | 城市名、中文全拼、citycode 或 adcode |
| `citylimit` | bool | 否 | 是否仅返回指定城市数据 |
| `children` | int | 否 | 是否按层级展示子 POI：0 不展示，1 展示 |
| `offset` | int | 否 | 每页记录数，默认 20，建议不超过 25 |
| `page` | int | 否 | 当前页码，默认 1 |
| `extensions` | string | 否 | 返回结果控制：`base` 返回基本信息，`all` 返回完整数据，默认 `base` |

适合场景：
- 搜索博物馆、公园、市场、画廊、书店、茶馆、餐厅、街区、酒店、商圈。
- 在某城市建立候选 POI 池。
- 按慢旅游主题搜索，例如老街、菜市场、公园、画廊、书店、咖啡、茶馆、河岸、社区街巷。

默认策略：
- 已知城市时传 `city`。
- 城市旅行规划中通常设置 `citylimit=true`。
- 使用较小 `offset`，拿到候选后筛选，不要把大量 POI 原样抛给用户。

### `amap_poi_around_search`

用于确定主锚点后查找附近补点和实用设施，是慢旅游路线里非常重要的工具。

参数：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `location` | string | 是 | 中心点坐标 `lng,lat`，例如 `116.481488,39.990464` |
| `keywords` | string | 否 | 搜索关键词，例如"餐厅"、"咖啡"、"亲子"、"地铁站" |
| `types` | string | 否 | POI 类型编码或类型名称，多个用 `\|` 分隔 |
| `city` | string | 否 | 城市名、citycode 或 adcode |
| `radius` | int | 否 | 查询半径（米），默认 3000；慢游建议 800–1500 |
| `sortrule` | string | 否 | 排序规则：`distance` 按距离排序，`weight` 综合评分排序，默认 `distance` |
| `offset` | int | 否 | 每页记录数，默认 20，建议不超过 25 |
| `page` | int | 否 | 当前页码，默认 1 |
| `extensions` | string | 否 | 返回结果控制：`base` 或 `all`，默认 `base` |

适合场景：
- 博物馆附近找午餐。
- 河岸散步附近找咖啡或茶馆。
- 景点结束后找最近地铁站。
- 找亲子设施、商场、洗手间、书店、展馆、公园。
- 为雨天、疲惫、人多时准备附近备选。

默认策略：
- 步行慢游半径建议 800-1500 米。
- 想减少绕路时用 `sortrule=distance`。
- 想兼顾质量/热度时可用 `sortrule=weight`，但不能盲目选择高人流热门点。

### `amap_poi_detail`

用于已知 POI ID 时确认地点详情。

参数：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | 是 | 高德 POI ID（可从搜索、输入提示结果中获取） |
| `extensions` | string | 否 | 返回结果控制：`base` 或 `all`（含更多详情），默认 `base` |

适合场景：
- 对搜索结果进行消歧。
- 确认从输入提示或关键词搜索得到的具体地点。
- 为行程条目绑定稳定 POI 身份。

注意：如果没有 POI ID，不要先调用它；先用输入提示或关键词搜索。

### `amap_geocode`

用于把结构化地址、酒店、车站、街区或地名转换为经纬度。

参数：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `address` | string | 是 | 结构化地址，例如"上海外滩"、"成都宽窄巷子" |
| `city` | string | 否 | 指定查询城市，可提高匹配精度、避免歧义 |

适合场景：
- 用户提供酒店名称或酒店地址。
- 用户提供出发点/终点。
- 路线和距离工具需要坐标。

注意：地址或地名有歧义时，应结合 `city` 参数和上下文判断。不要省略 `city` 参数来猜地点。

### `amap_regeocode`

用于把坐标反查成地址、行政区、附近 POI/AOI 等上下文。

参数：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `location` | string | 是 | 坐标 `lng,lat`，例如 `116.481488,39.990464` |
| `poitype` | string | 否 | 逆地理附近 POI 类型过滤，多个用 `\|` 分隔 |
| `radius` | int | 否 | 搜索半径（米），默认 1000 |
| `extensions` | string | 否 | 返回结果控制：`base` 或 `all`，默认 `base` |
| `roadlevel` | int | 否 | 道路等级：0 显示所有道路，1 过滤非主干道路 |
| `homeorcorp` | int | 否 | 附近 POI 优先策略：0 不干扰，1 偏居家，2 偏公司 |

适合场景：
- 判断一个坐标位于哪个区县或街区。
- 验证某个锚点周围是否适合形成慢游聚类。
- 解释为什么某几个 POI 可以放在同一天。

### `amap_district_search`

用于标准化城市、区县、adcode 和行政范围。

参数：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `keywords` | string | 否 | 行政区关键词，支持城市名、区县名或 adcode |
| `subdistrict` | int | 否 | 下级行政区级数：0 不返回下级，1 返回一级下级，默认 1 |
| `page` | int | 否 | 页码，默认 1 |
| `offset` | int | 否 | 每页记录数，默认 20 |
| `extensions` | string | 否 | 返回结果控制：`base` 或 `all`，默认 `base` |
| `filter` | string | 否 | 行政区划过滤条件，例如 `100000` |

适合场景：
- 城市或区域名称有歧义。
- 跨区、跨城、城市群路线规划。
- 搜索时需要限定城市或区县范围。

### `amap_ip_location`

仅在用户没有给出城市时，用来粗略推断默认城市。结果低置信度，不可用于精确旅行规划。

参数：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `ip` | string | 否 | 待定位 IP；为空时高德会尝试使用请求来源 IP |

适合场景：
- 用户说"周末附近去哪儿慢慢逛"，但完全没给城市。

注意：必须在输出中说明这是粗略默认城市。

## 距离与路线工具

### `amap_distance`

用于快速排序和聚类，通常在详细路线验证之前使用。

参数：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `origins` | []string | 是 | 起点坐标列表，每个值格式 `lng,lat`，例如 `[116.481488,39.990464]` |
| `destination` | string | 是 | 终点坐标 `lng,lat` |
| `type` | int | 否 | 测量类型：0 直线距离，1 驾车导航距离，3 步行规划距离，默认 0 |

适合场景：
- 比较多个候选 POI 到酒店、车站或锚点的距离。
- 识别明显绕路或离群的 POI。
- 判断某个候选点是否适合放进同一天。

类型建议：
- `0`：直线距离，用于早期粗聚类。
- `1`：驾车距离，用于自驾/包车行程。
- `3`：步行距离，用于街区慢游判断。

### `amap_route_walking`

用于短距离、人尺度的步行路线验证。

参数：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `origin` | string | 是 | 起点坐标 `lng,lat` |
| `destination` | string | 是 | 终点坐标 `lng,lat` |

适合场景：
- 两个相邻 POI 之间的风景步行。
- 地铁站到景点。
- 街区内慢游。
- 博物馆到咖啡店/餐厅。

慢旅游策略：
- 优先保留"步行本身也是体验"的路段。
- 如同行者有儿童、老人或行李，避免过长步行。

### `amap_route_transit`

用于地铁、公交等公共交通规划。

参数：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `origin` | string | 是 | 起点坐标 `lng,lat` |
| `destination` | string | 是 | 终点坐标 `lng,lat` |
| `city` | string | 是 | 起点所在城市，支持城市名、citycode 或 adcode |
| `cityd` | string | 否 | 终点所在城市，跨城公交时填写 |
| `strategy` | int | 否 | 公交换乘策略：0 最快捷，1 最经济，2 最少换乘，3 最少步行，5 不乘地铁，默认 0 |
| `nightflag` | int | 否 | 是否计算夜班车：0 否，1 是 |
| `date` | string | 否 | 出发日期，格式 `YYYY-MM-DD` |
| `time` | string | 否 | 出发时间，格式 `HH:mm` |
| `extensions` | string | 否 | 返回结果控制：`base` 或 `all`，默认 `base` |

适合场景：
- 城市自由行。
- 预算敏感路线。
- 酒店到目标街区。
- 不适合步行的跨区移动。

用户给出日期和时间时，应传入 `date` 和 `time` 以获得实时时刻表。

### `amap_route_driving`

用于自驾、包车、郊区景点和多途经点路线。

参数：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `origin` | string | 是 | 起点坐标 `lng,lat` |
| `destination` | string | 是 | 终点坐标 `lng,lat` |
| `originid` | string | 否 | 起点 POI ID，可提升路线准确性 |
| `destinationid` | string | 否 | 终点 POI ID，可提升路线准确性 |
| `strategy` | int | 否 | 驾车路线策略（0–20），默认 0（速度优先） |
| `waypoints` | []string | 否 | 途经点坐标列表，每个值格式 `lng,lat` |
| `avoidpolygons` | []string | 否 | 避让区域多边形坐标串列表 |
| `avoidroad` | string | 否 | 避让道路名称 |
| `province` | string | 否 | 车牌省份简称，用于限行判断 |
| `number` | string | 否 | 除省份外的车牌号码，用于限行判断 |
| `cartype` | int | 否 | 车辆类型：0 普通燃油，1 纯电动，2 插电混动，默认 0 |
| `ferry` | int | 否 | 是否使用轮渡：0 可以，1 不可以 |
| `roadaggregation` | bool | 否 | 是否将连续相同名称路段聚合 |
| `extensions` | string | 否 | 返回结果控制：`base` 或 `all`，默认 `base` |

适合场景：
- 亲子游、老人出行。
- 自然/郊区景点。
- 一天多个分散点。
- 携带行李或希望减少换乘。

策略：
- 使用 `waypoints` 测试拟定顺序是否合理。
- 若路线明显绕，应调整途经点顺序或删减远点。
- 有当地车牌时传入 `province` + `number` 获取准确的限行规避路线。

### `amap_route_bicycling`

用于适合骑行的慢游路线。

参数：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `origin` | string | 是 | 起点坐标 `lng,lat` |
| `destination` | string | 是 | 终点坐标 `lng,lat` |

适合场景：
- 河岸路线。
- 公园、校园、绿道。
- 低强度城市探索。
- 短距离风景走廊。

注意：骑行方案要考虑用户体力、天气和城市骑行安全感。

## 可视化工具

### `amap_static_map`

用于路线确定后生成静态地图 URL，展示路线或 POI 分布。

参数：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `location` | string | 否 | 地图中心点坐标 `lng,lat`；location/markers/paths 至少填写一个 |
| `zoom` | int | 否 | 缩放级别，通常 1–17，值越大越精细 |
| `size` | string | 否 | 图片尺寸，格式 `宽*高`，例如 `750*500` |
| `scale` | int | 否 | 图片精度：1 普通，2 高清（Retina），默认 1 |
| `markers` | string | 否 | 标注规则，按高德 staticmap markers 参数格式传入 |
| `labels` | string | 否 | 标签规则，按高德 staticmap labels 参数格式传入 |
| `paths` | string | 否 | 路线或区域规则，按高德 staticmap paths 参数格式传入 |
| `traffic` | bool | 否 | 是否叠加实时路况 |
| `validate` | bool | 否 | 是否实际请求 URL 验证状态码和内容类型；默认 `false` 只返回脱敏 URL |

markers 格式示例：`mid,0xFF0000,A:116.427,39.904;mid,0x0000FF,B:116.397,39.908`
paths 格式示例：`10,0x3366FF,1,,:116.427,39.904;116.397,39.908`

适合场景：
- 展示每日路线图。
- 展示同一天 POI 的地理聚类。
- 解释为什么路线顺路。

默认策略：
- 每天一张图优于多天混在一张图。
- marker 编号应与行程顺序一致。
- `paths` 只在能帮助理解路线时使用。
- 自驾或包车路线可按需叠加 `traffic=true`。
- 仅在需要确认图片有效性时设置 `validate=true`。
